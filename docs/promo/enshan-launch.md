# 标题
开源自部署：一个 LLM agent 帮你把想看的影视转存进自己的 115 / 夸克 / 光鸭网盘并回读验证（桌面版双击即用，也能上软路由 / NAS）

# 正文

自己折腾的一个开源项目，软路由 / NAS / 网盘折腾党应该对胃口，发上来分享 + 求反馈。我自己就把它跑在 iStoreOS 软路由上，24 小时挂着追更。

**一句话：** 你说要看哪部电影 / 剧 / 番，一个 LLM agent 跨资源站把最合适的资源**转存**进你自己的 115 / 夸克 / 光鸭网盘（秒传 / 离线，不下载到本地），转存后**回读网盘验证**到底落了什么，并持续追踪还缺哪几集。

- 🖥 **桌面版下载（Mac / Windows，双击即用）：** https://mediaryscout.app
- 🔗 **在线 demo（只读、免安装即玩）：** https://demo.mediaryscout.app
- 📦 **源码：** https://github.com/fancydirty/mediary-scout

demo 是纯前端回放，不连真盘、不写库，点进去直接看 agent 走「搜索 → 转存 → 验证 → 入库」整条流程。

## 两种用法

- **想先试试 / 单机用：** 官网下桌面版，双击打开，设置里填网盘和 LLM key 就能用。不用碰命令行、不用装 docker。
- **想 24 小时挂着追更 / 全家合用：** 软路由或 NAS 上 `docker compose up -d` 一把起（web + Postgres + 自带 PanSou 资源源），公网走 Cloudflare Tunnel，免公网 IP。这是我自己的用法。

## 解决什么痛点

大多数「媒体自动化」两种毛病：要么搜得好但根本不知道你到底缺哪集，要么会搬文件却从不验证自己落了什么。这东西把「获取」当成一个**状态问题**，由一个**凭证据而不是凭感觉**行动的 agent 驱动：

- **agent 选片**：读真实搜索结果，按画质偏好、**中文字幕**硬需求、去重来挑——没合格的宁可报「未找到」也不塞垃圾给你。转存后回读网盘核对真实落地文件。
- **追更 + 定时巡检**：季级状态机记着每部剧每季缺哪几集，定时只回来处理「仍有缺」的，不重复劳动。
- **多盘 / 多账号**：一个账号挂多块盘，每块盘都是一等工作区；一个实例还能全家合用——各自注册、各绑各的盘、各看各的库。
- **全程可观测**：活动页实时队列 + agent 每步动作 ticker；转存失败如实报「转存失败 + 原因」，不糊弄成「找不到资源」。

## 关于资源源（说清楚，别踩坑）

资源搜罗走 PanSou（可选叠加 Prowlarr）。**能搜到多好的资源，取决于 PanSou 连了哪些频道**——桌面版本身不含 PanSou 容器，想要更丰富的资源源，按官网教程自建一个（Mac 用 OrbStack / Win 用 Docker Desktop，几步的事）；docker 自部署那套则自带一个 PanSou。仓库里也附了一份好频道配置。

## 诚实定位

开源、纯自部署，**不提供也永远不会提供任何托管服务**——你自己跑实例、自带网盘 / LLM（任意 OpenAI 兼容端点）/ 元数据凭证，作者看不到你的任何东西。它做的就是你本来在自己网盘里也能手动完成的那些文件操作。
