# 标题
分享个开源自部署项目：LLM agent 帮你把想看的影视转存进自己的网盘（115 / 夸克 / 光鸭）并回读验证，桌面版双击即用 / 也能 docker 上机

# 正文

自己写的开源项目，VPS / 自部署 / 网盘折腾党应该有兴趣，分享一下顺便求反馈。

**一句话：** 你说要看哪部电影 / 剧 / 番，一个 LLM agent 跨资源站把最合适的资源**转存**进你自己的 115 / 夸克 / 光鸭网盘（秒传 / 离线，不下载到本地），转存后**回读网盘验证**到底落了什么，并持续追踪还缺哪几集。

- 🖥 **桌面版（Mac / Windows，双击即用）：** https://mediaryscout.app
- 🔗 **在线 demo（只读、免安装即玩）：** https://demo.mediaryscout.app
- 📦 **源码：** https://github.com/fancydirty/mediary-scout

demo 纯前端回放，不连真盘、不写库，点进去就能看 agent 怎么走「搜索 → 转存 → 验证 → 入库」。

## 为什么做这个

市面上的「媒体自动化」大多两种毛病：要么搜得好但根本不知道你到底缺哪集，要么会搬文件却从不验证自己落了什么。我把「获取」当成一个**状态问题**——一个**凭证据而不是凭感觉**行动的 agent：

- **搜 → 点获取 → agent 接管**：跨 PanSou / Prowlarr 搜罗，按画质偏好、**中文字幕**需求、去重挑片，转存后回读网盘核对真实落地文件。没合格的宁可报「未找到」也不塞垃圾。
- **追更 + 定时巡检**：季级状态机，定时只回来处理「仍有缺集」的剧。
- **多盘、多账号**：一个账号挂多块盘，每块盘都是一等工作区；一个实例还能多人合用，各绑各的盘、各看各的库。
- **全程可观测**：活动页实时队列 + agent 每步动作 ticker，不是黑箱转圈。

## 上机

- **单机 / 先试试：** 官网下桌面版双击就用，不碰命令行。
- **挂 VPS / NAS 24h 追更：** `docker compose up -d` 一把起（web + Postgres + 自带 PanSou），公网走 Cloudflare Tunnel，免公网 IP。网盘 / LLM key（任意 OpenAI 兼容端点）/ TMDB 全自带，作者看不到你任何凭证。

彩蛋：README 里放了段部署提示词，丢给 Claude Code / Codex / opencode，agent 会按问答引导帮你把实例部署起来。

## 关于资源源

搜罗走 PanSou（可选叠加 Prowlarr）。能搜到多好的资源取决于 PanSou 连了哪些频道——桌面版不含 PanSou 容器（想要更丰富的源按官网教程自建，Mac OrbStack / Win Docker Desktop）；docker 那套自带一个。仓库附了好频道配置。

## 诚实定位

开源、纯自部署，**不提供也永远不会提供托管服务**——你自己跑实例、自带网盘 / LLM / 元数据凭证。它做的就是你本来在自己网盘里也能手动完成的那些文件操作。
