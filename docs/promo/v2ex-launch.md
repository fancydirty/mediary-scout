# 正文

自己折腾的一个开源项目，分享一下，顺便求点反馈。

一句话：**你说要看哪部电影 / 剧 / 番，一个 LLM agent 跨资源站把最合适的资源转存进你自己的 115 / 夸克 / 光鸭网盘，转存后回读验证，并持续追踪还缺哪几集。**

- **桌面版（Mac / Windows，双击即用）：** https://mediaryscout.app
- **在线 demo（只读、免安装即玩）：** https://demo.mediaryscout.app
- **源码：** https://github.com/fancydirty/mediary-scout

demo 是纯前端回放真实流程，不连任何真盘、不写库，点进去就能看 agent 怎么干活。

## 为什么做这个

市面上的「媒体自动化」大多两种毛病：要么搜得好但根本不知道你到底缺哪集，要么会搬文件却从不验证自己落了什么。我把「获取」当成一个**状态问题**来做——一个**凭证据而不是凭感觉**行动的 agent：

- **搜 → 点「获取」→ agent 接管**：跨 PanSou / Prowlarr 搜罗，按画质偏好、**中文字幕**需求、去重来挑片，转存（秒传 / 离线，不往本地磁盘下载）后**回读网盘验证**到底落了什么。
- **追更 + 定时巡检**：季级状态机，定时只回来处理「仍有缺集」的剧。
- **多盘、多账号**：一个账号挂多块盘，每块盘都是一等工作区（树模型）；一个实例还能多人合用，各绑各的盘、各看各的库。
- **全程可观测**：活动页有实时队列 + agent 每一步动作的 ticker，不是黑箱转圈。

## 两种用法

- **单机 / 先试试：** 官网下桌面版双击就用，不碰命令行、不装 docker，数据存你自己电脑。
- **挂 24h 追更 / 全家合用：** `docker compose up -d` 一把起（web + Postgres + 自带 PanSou 资源源），公网走 Cloudflare Tunnel（免公网 IP）。网盘 / LLM key（任意 OpenAI 兼容端点）/ TMDB 全自带，作者看不到你的任何凭证。

有个我自己觉得挺好玩的点：README 里放了段提示词，你把它丢给 Claude Code / Codex / opencode，**让 agent 按问答引导帮你把实例部署起来**。

## 诚实定位

开源、纯自部署软件，**不提供、也永远不会提供任何托管服务**——你自己跑实例、自带网盘 / LLM / 元数据凭证。它做的就是你本来在自己网盘里也能手动完成的那些文件操作。

技术栈是 Next.js（App Router / PPR）+ 进程内 worker 驱动的沙盒 agent + Postgres 队列与状态 + Vercel AI SDK。一个人写的，欢迎拍砖、提 issue / PR，或者就去 demo 点两下告诉我哪里别扭。
